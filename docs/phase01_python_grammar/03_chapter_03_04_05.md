# Chapter 3. 매직 메서드 (Dunder Methods)와 연산자 오버로딩

파이썬의 가장 강력한 무기 중 하나인 던더(`__`, double underscore) 메서드를 배웁니다. 이를 통해 우리가 만든 `Value` 객체를 덧셈 기호(`+`)나 곱셈 기호(`*`)로 마치 일반 숫자처럼 연산할 수 있게 만들어 딥러닝 수식을 매우 직관적으로 작성할 수 있습니다.

## 3-1. 매직 메서드 개념과 문자열 표현(`__repr__`)
파이썬은 예약된 특수 메서드 이름(예: `__init__`, `__add__`)을 사용하여 특별한 구문(예: 인스턴스 생성, 연산자 사용)에 대해 내부적으로 어떻게 동작할지 정의할 수 있습니다.

```python
class Value:
    def __init__(self, data):
        self.data = data
        
    # __repr__: 객체를 출력(print)할 때 예쁘게 문자열로 보여주는 매직 메서드
    def __repr__(self):
        return f"Value(data={self.data})"

a = Value(5.0)
print(a) # 출력: Value(data=5.0)
```
**핵심 포인트**: `microgpt.py` 원본 코드에는 코드를 최소화하기 위해 `__repr__`가 생략되어 있으나, 디버깅을 위해서는 이처럼 출력을 정의해 두는 것이 좋습니다.

## 3-2. 사칙연산 오버로딩 1 (기본 연산)
파이썬에서 `c = a + b`를 실행하면 인터프리터는 내부적으로 `c = a.__add__(b)`를 호출합니다. 이 원리를 이용해 `+`, `*` 연산자가 작동하도록 오버로딩(Overloading: 재정의)합니다.

```python
class Value:
    def __init__(self, data):
        self.data = data
        
    # __add__: 덧셈 (+) 오버로딩
    def __add__(self, other):
        # 1. 두 객체의 data를 더함
        new_data = self.data + other.data
        # 2. 더한 결과를 가진 완전히 새로운 Value 객체를 반환!
        return Value(new_data)
        
    # __mul__: 곱셈 (*) 오버로딩
    def __mul__(self, other):
        return Value(self.data * other.data)

a = Value(2.0)
b = Value(3.0)
c = a + b  # a.__add__(b) 로 자동 치환됨
d = a * b  # a.__mul__(b) 로 자동 치환됨

print(c.data) # 5.0
print(d.data) # 6.0
```
**핵심 포인트**: 이 챕터의 핵심은 **`a + b`를 했을 때 기존의 값을 바꾸는 것이 아니라, "새로운 인스턴스를 만들어서 반환(return)한다"**는 점입니다. 이로 인해 계산 과정상에 탄생하는 모든 중간 값들이 `Value`라는 껍데기를 입은 채 계산 그래프 상의 독립적인 노드(Node)로 편입되게 됩니다.

## 3-3. 단항 연산자와 거듭제곱 (`__neg__`, `__pow__`)
마찬가지로 `-a`, `a**2`와 같은 연산자들도 재정의할 수 있습니다.

```python
class Value:
    def __init__(self, data):
        self.data = data
    
    # __neg__: 부호 반전 (-a)
    def __neg__(self):
        return Value(-self.data)
        
    # __pow__: 거듭 제곱 (a ** other)
    def __pow__(self, other):
        # other가 숫자로 들어온다고 가정
        return Value(self.data ** other)

a = Value(3.0)
print((-a).data)   # -3.0
print((a**2).data) # 9.0
```


# Chapter 4. 매직 메서드 심화 및 타입 대응

실제 딥러닝 코드를 작성하다 보면 `a = a + 1.0` 처럼 우리가 만든 객체와 단순 숫자가 섞여 연산되는 경우가 매우 많습니다. 이를 에러 없이 부드럽게 넘기는 파이썬의 타입 회피 기술들을 다룹니다.

## 4-1. 타입에 따른 분기 처리 (`isinstance`)
덧셈 과정에서 상대방(`other`)이 `Value` 객체면 그대로 `other.data`를 쓰면 되지만, 단순 숫자(`int`나 `float`)가 들어왔다면 `Value` 객체로 한 번 감싸줘야(Wrap) 동등한 연산이 성립합니다.

```python
class Value:
    def __init__(self, data):
        self.data = data
        
    def __add__(self, other):
        # other가 Value 클래스의 인스턴스(객체)인지 확인
        # 객체가 아니라면 Value(other)로 씌워줌 (Wrap)
        other = other if isinstance(other, Value) else Value(other)
        
        # 이제 other.data를 안심하고 접근할 수 있음
        return Value(self.data + other.data)

a = Value(2.0)
b = a + 3.0   # 3.0은 객체가 아니지만 분기문에 의해 Value(3.0)으로 변환됨!
print(b.data) # 5.0
```
**핵심 포인트**: `microgpt.py`의 `__add__`와 `__mul__` 상단에 들어간 형태가 정확히 이 역할을 수행합니다. 이를 통해 매우 유연한 방정식 작성이 가능해집니다.

## 4-2. 우항 연산자(Right-side operations) 다루기
그렇다면 반대로 `3.0 + a` 형태는 어떨까요?
파이썬은 먼저 숫자 `3.0`에게 `__add__(a)`를 시도하지만 실패합니다(`3.0`은 파이썬 내장 객체이므로 어떻게 `Value`를 다뤄야 할지 모름). 이 경우 파이썬은 두 번째 피연산자인 타겟 객체(`a`)에게 "혹시 우측에서 덧셈 당할 수 있니?" 라며 `__radd__`(Right-Add) 규칙이 구현되어 있는지 묻게 됩니다.

```python
class Value:
    def __init__(self, data):
        self.data = data
        
    def __add__(self, other):
        other = other if isinstance(other, Value) else Value(other)
        return Value(self.data + other.data)
        
    # __radd__: 다른 타입(숫자) + Value 연산 시 호출됨
    def __radd__(self, other):
        # 교환 법칙이 성립하므로 그냥 self + other로 자리를 바꿔서 원래 __add__를 호출시킴!
        return self + other
        
    # __mul__ 과 __rmul__ 도 우항 연산 교환 법칙이 성립
    def __mul__(self, other):
        other = other if isinstance(other, Value) else Value(other)
        return Value(self.data * other.data)
        
    def __rmul__(self, other):
        return self * other

a = Value(2.0)
b = 3.0 + a # 에러 없이 3.0의 __add__ 대신 a의 __radd__ 가 작동하여 5.0이 됨
```
**핵심 포인트**: 
덧셈과 곱셈은 순서를 바꿔도 결과가 같기 때문에 `return self + other`처럼 위임(Fallback)하면 됩니다.
하지만 뺄셈(`a - 3.0` vs `3.0 - a`)과 나눗셈은 순서 교환 법칙이 성립하지 않습니다! 
때문에 `microgpt.py`의 `__rsub__`는 `return other + (-self)`로, `__rtruediv__`는 `return other * self**-1` 과 같은 수학적 공식을 빌려 안전하게 해결했습니다.


# Chapter 5. 고급 파이썬 활용 기술

마무리로 트리를 제어하고 메모리의 물리적 한계를 극복하는 두 가지 필수 최적화 기술입니다.

## 5-1. 파이썬 메모리 최적화와 `__slots__`
보통 파이썬으로 클래스를 선언하면, 내부 변수는 딕셔너리(`__dict__`)라는 자료구조에 자유롭게 묶입니다. 딕셔너리는 확장이 자유로운 대신 오버헤드가 크고 메모리를 낭비합니다. GPT같은 딥러닝 망에서는 `Value` 객체가 수십, 수백만 개가 생겨나므로 메모리 절약이 필수입니다.

```python
class Value:
    # __slots__: 클래스가 가질 변수를 고정된 튜플 형태로 선언 (딕셔너리 자동 생성을 차단)
    __slots__ = ('data', 'grad', '_children', '_local_grads')

    def __init__(self, data):
        self.data = data
        self.grad = 0
```
**핵심 포인트**: 위처럼 미리 슬롯(`__slots__`)을 지정해두면, 파이썬이 해시 테이블(Dict)을 유지할 책임을 덜어 각 `Value` 객체의 크기가 수십 분의 1로 획기적으로 줄어듭니다.

## 5-2. 재귀 호출 (Recursion)과 DFS 분기 탐색
`microgpt.py`의 심장부인 `backward()` 과정에 쓰이는 위상 정렬은 "깊이 우선 탐색(DFS)" 이라는 알고리즘에 기초합니다. 이때 필수로 알아야 하는 패턴이 바로 `재귀(Recursion)` 함수입니다.

```python
def print_parents(node_name):
    print(f"{node_name} 탐색 중...")
    
    # 멈추는 조건이 없거나 부모가 없는 최상위 노드를 판별하지 못하면 무한루프에 빠짐!
    # 여기서는 간단히 자식이 한 단계라고 가정
    if node_name == "최상위 조상": 
        return
        
    # 자기 자신을 함수 내부에서 다시 호출 (재귀적 반복)
    print_parents("최상위 조상")

print_parents("자식 노드")
```
**핵심 포인트**: 우리가 만든 연산 그래프는 뿌리(`Loss`)에서 시작해서 나뭇가지(`a`, `b`, `c`)들로 끝없이 뻗어 나갑니다. 이를 탐색하려면 자기 자신의 로직(`자식 노드 찾아들어가기`)을 끝없이 반복해야 하며 파이썬의 재귀 성질이 복잡한 위상 정렬 과정을 아주 우아하게 해결해줍니다.
