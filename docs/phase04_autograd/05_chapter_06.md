# Chapter 6. 학습 루프와 Adam 옵티마이저

이제 `microgpt.py`의 대미를 장식하는 371~405줄, **학습 루프(Training Loop)**입니다. 지금까지 만든 모든 부품(Value 엔진, 정방향 모델)을 가동하여 모델이 스스로 지식을 쌓는 과정입니다.

## 6-1. 데이터 전처리 (토큰 세팅)

```python
for step in range(num_steps): # 1000번 반복 학습

    # 1. 문서 하나 가져오기 (예: "emma")
    doc = docs[step % len(docs)]
    
    # 2. 토큰 시퀀스 생성: BOS + 단어토큰 + BOS
    tokens = [BOS] + [uchars.index(ch) for ch in doc] + [BOS]
    # "emma" → [26, 1, 4, 4, 0, 26]
    
    # 3. 길이 제한 (안전장치)
    n = min(block_size, len(tokens) - 1)
```

**학습의 목표**: 현재까지 주어진 토큰들을 보고, **바로 다음 순서의 토큰이 무엇인지** 예측하는 것입니다. `[26]`을 보고 `1`을, `[26, 1]`을 보고 `4`를 예측해야 합니다.


## 6-2. 순전파와 크로스 엔트로피 손실

시퀀스의 맨 처음부터 한 칸씩 넘어가면서 해당 위치까지의 예측과 정답 비교를 전부 진행합니다. 즉, 글씨를 한 글자씩 써내려가는 과정 전체를 복기하며 채점합니다.

```python
loss_t = ... # 손실 초깃값 (생략)
losses = []

# 글자 수만큼 반복!
for pos_id in range(n):
    token_id = tokens[pos_id]     # 모델에 넣어줄 '현재' 글자 
    target_id = tokens[pos_id +1] # 우리가 맞혀야 할 '다음' 정답 글자
    
    # 트랜스포머 망 전체 통과 (순전파 진행, 계산 그래프 실시간 형성)
    logits = gpt(token_id, pos_id, keys, values)
    
    # 로짓을 확률로 변환 (Phase 2에서 배운 Softmax)
    probs = softmax(logits)
    
    # 정답 글자가 나올 확률값(probs[target_id])에 대해서만 손실 계산
    # (Phase 2에서 배운 -ln(p) Cross-Entropy Loss!)
    loss_t = -probs[target_id].log() 
    losses.append(loss_t)

 # 단어 구절 전체의 손실 평균내기
loss = (1 / n) * sum(losses)
```


## 6-3. 역전파 엔진 (Backward Pass)

가장 위대한 마법이 일어나는 순간입니다.
위에서 계산한 최종 `loss` 변수는 그동안 곱해지고 더해진 모든 `Value` 노드들의 거대한 트리 꼭대기에 있는 뿌리(Root)입니다.

```python
loss.backward()
```
단 한 줄의 코드로 `loss` 객체 내부에 있는 메소드가 실행되며 `build_topo()`가 호출되고, DFS를 돌아 위상 정렬을 끝낸 후 방금 일어난 수천수만 번의 연산을 거꾸로 헤집고 내려가기 시작합니다!

그 결과, 평탄화해 두었던 뇌세포 리스트 `params` 안의 모든 노드들이 "내가 loss에 직간접적으로 이만큼 기여했구나"를 뜻하는 `.grad` 속성에 값을 채워넣게 됩니다.


## 6-4. Adam 옵티마이저 상세

옵티마이저(Optimizer)는 알아낸 기울기(`.grad`)를 이용해 파라미터(`.data`)를 어떻게 바꿀지 결정하는 업데이트 엔진입니다. 

단순한 경사하강법(SGD)은 속도가 의미하는 기울기를 그대로 더하지만(`data -= lr * grad`), 최고 효율 알고리즘인 **Adam**은 **관성(가속도)**의 개념을 추가했습니다.

### Adam의 핵심 원리
1. **첫 번째 모멘트 (`m`)**: 방금 구해진 기울기(`grad`)들의 연속된 이동 평균입니다. 즉, 예전 이동하던 방향(관성)을 계속 유지하려는 습성입니다.
2. **두 번째 모멘트 (`v`)**: 기울기의 **크기(제곱)**에 대한 평균입니다. 내가 지금 얼마나 크게 요동치고 있는지를 기억합니다.
3. **편향 보정 (`m_hat`, `v_hat`)**: 초기 스텝에서는 `m`과 `v`가 지나치게 0에 가까워지기 때문에 보정해 주는 수학적 안전장치입니다.

```python
for i, p in enumerate(params):
    # 1. 관성 기록 (예전 관성 85% + 현재 기울기 15%)
    m[i] = beta1 * m[i] + (1 - beta1) * p.grad
    
    # 2. 크기 기록 (예전 크기 99% + 현재 기울기 제곱 1%) 
    v[i] = beta2 * v[i] + (1 - beta2) * p.grad ** 2
    
    # 3. 보정 (0이 되는 것 방지)
    m_hat = m[i] / (1 - beta1 ** (step + 1))
    v_hat = v[i] / (1 - beta2 ** (step + 1))
    
    # 4. 실질 업데이트!
    # 기울기의 방향(m)을 기울기 요동 크기(√v)로 나눔
    # (변동폭이 컸던 놈은 보수적으로 덜 움직이게 됨)
    p.data -= lr_t * m_hat / (v_hat ** 0.5 + eps_adam)
    
    # 5. [매우 중요] 다음 스텝을 위해 이번 턴의 기울기 누적치를 다시 0으로 청소
    p.grad = 0
```
마지막 `p.grad = 0` (Zero Grad)을 하지 않으면, 이전 스텝에서 계산한 기울기에 더해서 미분값이 끝없이 누적합(`+=`) 되어버려 학습이 완전히 파괴됩니다!


## 6-5. 학습률 스케줄링

학습률(Learning Rate)은 모델 보폭의 크기입니다. 거대한 오차의 산백에서 내려갈 때, 초반에는 방향만 맞으면 성큼성큼 크게 전진하고, 산기슭(정답 근처)에 다다르면 보폭을 좁혀야 정확한 타겟에 도달할 수 있습니다.

```python
# 현재 진행도(step / num_steps)에 비례해서
# 초기(0.01)에서 루프의 끝(0.0)까지 일정하게 줄어드는 선형 감쇠
lr_t = learning_rate * (1 - step / num_steps)
```
