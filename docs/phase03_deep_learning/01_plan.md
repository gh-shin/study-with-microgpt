# Phase 3: 딥러닝 코어 이론 - 계산 그래프와 역전파

이 단계에서는 앞서 배운 수학 연산과 파이썬 문법을 합쳐서, `microgpt.py`가 알고리즘 자료구조를 통해 실제로 어떻게 스스로 미분값을 찾아가는지 배웁니다.

## 📌 [Chapter 1. 계산 그래프 (Computation Graph)](02_chapter_01_02.md)
수식을 컴퓨터가 이해할 수 있는 트리(Tree) 형태의 방향성 비순환 그래프(DAG) 구조로 변환하는 과정을 배웁니다.
* **1-1. 코드가 그리는 그래프**: `Value` 객체의 `_children` 튜플이 엮여서 만들어지는 족보 가계도
* **1-2. DAG (Directed Acyclic Graph)**: 방향이 있고 순환하지 않는 그래프 구조. 왜 순환하면 안 되는가?

## 📌 [Chapter 2. 순전파 (Forward Pass)](02_chapter_01_02.md#chapter-2-순전파-forward-pass)
입력 데이터(`input`)에서 출발하여 최종 손실(`loss`)까지 계산이 뻗어나가는 과정입니다.
* **2-1. 평가와 결과물**: 연산을 수행하여 각 노드의 `self.data`를 확정 짓는 단계
* **2-2. 그래프의 암묵적 완성**: 순전파가 진행될 때 동시에 `_children` 관계가 완성되며 빈 도화지에 지도가 그려지는 원리

## 📌 [Chapter 3. 위상 정렬 (Topological Sorting)](03_chapter_03_04_05.md)
아무렇게나 역전파를 하면 안 됩니다. 부모의 그래디언트가 준비되어야만 내 그래디언트를 계산할 수 있습니다.
* **3-1. 줄 세우기 알고리즘**: 복잡한 트리 구조의 작업들을 일직선으로(순서대로) 정렬하는 기법
* **3-2. `build_topo` 함수 분석**: DFS(깊이 우선 탐색)와 Set을 이용한 중복 방지 로직 분석

## 📌 [Chapter 4. 역전파 (Backward Pass) 알고리즘](03_chapter_03_04_05.md#chapter-4-역전파-backward-pass-알고리즘)
위상 정렬이 끝났다면, 이제 정렬된 리스트를 거꾸로 돌면서 미분값을 곱해 나가는 딥러닝의 핵심 체인 룰을 실행합니다.
* **4-1. 역전파 엔진 (`backward`)**: 순번을 거꾸로 돌며(`reversed`) 자식 노드들에게 글로벌 그래디언트를 분배하는 `microgpt.py`의 핵심 루프문 탐구
* **4-2. 루트 노드의 초기화**: 최초로 출발하는 `loss.grad = 1` 의 의미

## 📌 [Chapter 5. 활성화 함수와 비선형성 (ReLU)](03_chapter_03_04_05.md#chapter-5-활성화-함수와-비선형성-relu)
직선만으로는 복잡한 뇌를 모방할 수 없습니다. 곡선을 만들어주는 장치인 활성화 함수에 대해 알아봅니다.
* **5-1. 왜 비선형성이 필요한가?**: 선형 연산($\times$, $+$)만 겹쳤을 때 선형으로 다시 함몰되는 현상 방지
* **5-2. `relu()` 의 동작과 미분**: 0보다 작으면 완전히 죽여버리고(0), 크면 그대로 통과(1배)시키는 ReLU의 파이썬 코드(`float(self.data > 0)`)와 그 로컬 미분에 대한 해석

## 📌 [Chapter 6. 토큰화(Tokenization)와 임베딩(Embedding)](04_chapter_06.md)
텍스트 데이터가 신경망에 들어가기 위해 거치는 변환 과정을 배웁니다. 문자를 숫자로, 숫자를 벡터로 바꾸는 두 단계의 변환입니다.
* **6-1. 토큰화**: 문자열을 정수 시퀀스(토큰)로 변환하는 원리, `uchars` 사전과 `BOS` 특수 토큰의 역할
* **6-2. 토큰 임베딩 (Token Embedding)**: 정수 ID 하나를 실수 벡터로 변환하는 룩업 테이블(`wte`) 원리
* **6-3. 위치 임베딩 (Position Embedding)**: 순서 정보를 벡터에 주입하는 `wpe`와 왜 언어 모델에 위치가 중요한지

## 📌 [Chapter 7. 어텐션 메커니즘과 트랜스포머](05_chapter_07.md)
GPT의 두뇌인 트랜스포머 구조를 직관적으로 이해합니다. 수학에 서툴어도 Query, Key, Value의 비유를 통해 어텐션의 본질을 파악할 수 있습니다.
* **7-1. 어텐션(Attention)의 직관**: Query, Key, Value가 하는 역할을 도서관 검색 비유로 이해
* **7-2. 스케일드 닷 프로덕트 어텐션**: 점곱 → 스케일링(`/ √d`) → 소프트맥스 → 가중합의 4단계 흐름
* **7-3. 멀티 헤드 어텐션 (Multi-Head Attention)**: 왜 여러 개의 관점(Head)으로 나눠서 보는가
* **7-4. MLP (Feed-Forward) 블록**: 어텐션 이후 정보를 가공하는 2층 신경망의 역할
* **7-5. `gpt()` 함수 전체 흐름 정리**: 입력에서 로짓까지의 데이터 파이프라인

## 📌 [Chapter 8. 정규화(Normalization)와 잔차 연결(Residual Connection)](06_chapter_08.md)
깊은 신경망이 안정적으로 학습할 수 있도록 돕는 두 가지 핵심 기법을 배웁니다.
* **8-1. 왜 정규화가 필요한가**: 레이어를 지날수록 값이 폭발하거나 소멸하는 문제
* **8-2. RMSNorm 함수의 동작**: 평균 제곱근으로 벡터를 정규화하는 공식과 `epsilon(1e-5)` 안전장치
* **8-3. 잔차 연결 (Skip Connection)**: `x + f(x)` 패턴이 기울기 소실을 방지하는 원리
